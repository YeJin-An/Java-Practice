추상클래스란?

정의 -> 객체 생성 불가 클래스(타입)
          : 자식 클래스를 통해 간접적으로 객체 생성
형식 -> abstract키워드를 사용하여 추상 클래스를 선언


어떠한 객체가 스스로 존재할 수 없는 경우
-> 또는 존재하면 안되는 경우
-> 어떠한 객체가 스스로 존재할 수 없는 경우
-> abstract키워드를 사용하여 선언.


추상클래스의 장점
1. 상속의 장점
-> 코드 중복제거
2. 인터페이스의 장점
-> 다형성과 그룹화 


[abstract] 추상메서드란?
추상클래스를 상속받은 클래스는
추상클래스 안에 있는
모든 추상 메서드를 구현해야 한다.



인터페이스와 추상클래스의 차이점
인터페이스 -> 모조리 추상메서드
--> 설계도
--> 계약서
추상클래스 -> 추상메서드 + 일반메서드
--> 복제
--> 상속을 받아 기능을 확장하는데, 이미지가 있다.
